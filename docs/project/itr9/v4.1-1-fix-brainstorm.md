# generatedValue 修正計画

**日付**: 2025-12-08
**問題**: BrainstormCardsでsampleCardsが生成されない

---

## 問題分析

### 症状
- E2Eテストで`brainstorm_cards`の`sampleCards`が生成されない
- 一方、`tradeoff_balance`の`initialItems`（items）は生成されている

### 原因
プロンプトの**出力例セクション**（L111-201）でbrainstorm_cardsのconfig例が以下のようになっている:
```json
"config": {
  "title": "アイデアを出してみましょう",
  "placeholder": "思いついたことを書いてください",
  "minItems": 3,
  "maxItems": 10
}
```
→ **sampleCardsが含まれていない**

LLMは出力例を強く参照するため、generationHintsセクション（L71-106）の指示があっても出力例に従ってしまう。

### TradeoffBalanceが動作した理由
- `tradeoff_balance`には`generationHints`がない
- しかしWidget定義の`ports.inputs`に`items`があり、LLMがそれを見て`config.items`を生成した可能性
- または、既存のWidget実装パターン（SwotAnalysis等）でitemsを使う慣習がある

---

## 修正方針

プロンプトの出力例を更新し、brainstorm_cardsの例にsampleCardsを含める。

### Step 4: UISpec生成プロンプト更新

**ファイル**: `server/src/prompts/v4/uispec-generation.prompt.ts`

L166以降に追加:

```
### 5. コンテンツ生成（generatedValue）

Widget定義にgenerationHintsがある場合、ユーザーの悩みに関連するコンテンツを生成してください。

**Type B: サンプルデータ**
generationHints.samplesがある場合:
- fieldで指定されたキーにGeneratedContentContainerを配置
- instructionに従い、countで指定された数のアイテムを生成
- 各アイテムにisGenerated: trueを付与
- コンテナにもisGenerated: trueを付与

**例: BrainstormCards**
ユーザーの悩み: 「転職するか迷っている」
→ config.sampleCards:
{
  "items": [
    { "id": "sample_1", "text": "現職のメリット・デメリット", "isGenerated": true },
    { "id": "sample_2", "text": "転職で実現したいこと", "isGenerated": true }
  ],
  "isGenerated": true
}

**重要**:
1. 生成内容はユーザーの具体的な悩みに関連させる
2. サンプルは「考えるきっかけ」であり、完成した答えではない
3. 日本語で生成する
```

  setSampleCards(prev => prev.filter(s => s.id !== sample.id));
  emitAllPorts();
}, [emitAllPorts]);

}, []);
